3000 단어 pillar page. cluster 의 중심.
You are an SEO content writer for LuckyPlz, a 16-language decision games site.

Task: Write a 3000-word pillar page for the keyword "[KEYWORD]".

Context:
- Search intent: informational + commercial
- Target reader: [PERSONA, e.g., "casual users looking for free decision tools"]
- Reading level: 8th grade
- Tone: Friendly, expert, evidence-based

Structure:
1. Hook (150w) — Relatable problem, promise of answer. First 100w must include exact keyword.
2. Quick Answer (100w) — TL;DR for skimmers
3. Background (400w) — Why this matters, brief history
4. Main content (1800w) — 5-6 H2 sections covering:
   [SECTION 1], [SECTION 2], [SECTION 3], [SECTION 4], [SECTION 5]
5. Comparison table (150w + table) — vs alternatives
6. FAQ (350w) — 8 questions from "people also ask"
7. CTA (100w) — Link to /games/[X]/

Constraints:
- Include 3 statistics (real or noting "estimated based on industry data")
- 8-10 internal link suggestions in [brackets]
- 4 image placeholder descriptions
- Add FAQ schema markup at the end
- 5 LSI keywords: [list]
- Avoid AI clichés: delve, paramount, in conclusion, navigate, leverage, embark
- Use active voice, contractions OK

Output: Markdown with proper H1/H2/H3 hierarchy.
변수: [KEYWORD] · [PERSONA] · [SECTION 1-5] · [LSI keywords] · [X game]
P-02 · How-to 튜토리얼CORE
step-by-step 가이드. 검색량 큰 "how to X" 키워드.
Write a 1800-word how-to article for "How to [TASK]".

Structure:

3. Step 1: [STEP NAME] (250w) — Detailed instructions, screenshots placeholder
4. Step 2: [STEP NAME] (250w)
5. Step 3: [STEP NAME] (250w)
6. Step 4: [STEP NAME] (250w)
7. Common Mistakes (200w) — 5 pitfalls + how to avoid
8. Pro Tips (150w) — 3 advanced tricks

Each step:
- Number prominent
- "Before you do this..." (context)
